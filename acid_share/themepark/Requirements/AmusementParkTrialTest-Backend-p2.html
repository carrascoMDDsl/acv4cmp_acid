Welcome to the CM Productions AMUSEMENT PARK trial test (page 2/2)

This page is about what we expect from you. Read it carefully.

Your objective

  • You are requested to design the model of the MVC structure of the park, so other coders can build any application ontop of it.
  • The static data for the model comes from the DB
  • The real-time data for the model comes from an API
  • The output of your model is what we call "The Model API"
    • - You are the responsible of designing and creating that API.

The RealTime API

  • Not to be confused with "the Model API".
  • This API will be called by your Model to get real time data.
  • It is based on POST requests to a controller, sending JSON data and obtaining JSON data.
  • The API controller is this one: api.php
  • The API documentation is here: readme.txt
  • For you convenience we have built and API-TESTER taht we reccomend to try: apiTest.html - We have tested it in Firefox

Your API

  • You can imagine that the application that is going to consume your application can be either:
    • a controller from an HTTP request.
    • a command for the command line.
  • The controller, or the command line, will invoke you as this:
    • $parkManager = $serviceProvider->getService( "parkManager" );
$park = $parkManager->getPark( "SeaWorld" );
$park->.....

How we are going to evaluate you

  • We are NOT expecting that you write the code.
  • We expect that you:
    1. Write one text file listing the classes, interfaces, methods, etc. but only the signatures, not the content.
    2. If you prefer, you can draw a block-diagram (UML or any invented syntax can work)
  • What we valuate the most is your mind structure.
  • We therefore will be much happier if you think out loud even if you are having "temporary thoughts"
  • We also will evaluate how intelligent are your questions. We have not provided all the information and we seek a balance of people that is autonomous (so will not interrupt the team at each second) but also that do not get stuck to a problem for a long time and are able to request help. Do your best to be "an autonomous team player"
  • When you are investigating... we would love to hear what are you looking for and why.
  • We will valuate better clean code that is mainatainable over time, more than code that just works but lacks profound thinking.

Example of output

  • We expect that you write a text document with the API of the MODEL, listing the classes, the interfaces, the inheritances, etc, and for each one, the methods it's going to provide, with the return datatype and the parameters (and types) you will pass to each method
You can use a metalanguage similar to the following example: 
Interface MyInterface1

Class MyClass1 implements MyInterface1
    void method1( int $param1, double $param2 );

Class MyClass2 extends Whatever
    MyClass1 getMethod1( int $id );
